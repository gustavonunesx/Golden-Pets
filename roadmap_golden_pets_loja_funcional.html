Roadmap de Lançamento

Guia prático para colocar a Golden Pets no ar.

Progresso do Projeto 0%
1
Banco de dados e autenticação
Supabase — gratuito até 500MB · ~1 semana de trabalho
0/4 concluídos
1
Criar conta e projeto no Supabase
Seu banco de dados PostgreSQL gratuito na nuvem
Grátis ▼
Acesse supabase.com e crie conta com GitHub
Clique em New Project → dê o nome "golden-pets" → escolha região South America (São Paulo)
Anote as credenciais: Project URL e anon public key — você vai precisar no .env
# instalar no seu projeto Next.js
npm install @supabase/supabase-js @supabase/ssr
Crie o arquivo .env.local na raiz do projeto com as variáveis do Supabase
O plano gratuito inclui 500MB de banco, 5GB de storage e 50.000 usuários autenticados.
2
Criar tabelas do banco de dados
Produtos, pedidos, clientes, categorias
Crítico ▼
No painel do Supabase vá em SQL Editor e execute as queries para criar as tabelas.
3
Configurar autenticação de admin
Só você acessa o dashboard
Rápido ▼
No Supabase vá em Authentication → Users e crie seu usuário admin.
Crie um middleware.ts na raiz para proteger as rotas /admin/*.
4
Conectar front-end ao banco
Substituir mocks pelo banco real
Crítico ▼
Crie /lib/supabase.ts com o client configurado. Substitua consultas estáticas por queries Server Components.
2
Gateway de pagamento
MercadoPago — melhor opção para Brasil iniciante · ~1 semana
0/5 concluídos
1
Por que MercadoPago para começar
Comparativo dos gateways para o mercado brasileiro
Info ▼
Taxa ~4,49% (cartão 14 dias). API fácil, amplamente conhecido pelo consumidor BR.
2
Criar conta e obter credenciais
MercadoPago Developer — acesso à API
Grátis ▼
Vá em Seu negócio → Credenciais. Use as de teste primeiro.
3
Criar a rota de API de checkout
/api/checkout — recebe pedido e cria preferência
Crítico ▼
Receba o carrinho via POST, crie preferência e retorne o ID.
4
Webhook — confirmar pagamento
MercadoPago avisa seu site quando o pagamento é aprovado
Crítico ▼
Atenção: sem o webhook configurado, você nunca saberá se o pagamento foi aprovado no seu sistema.
5
Carrinho de compras no front-end
Estado global com Zustand
Front-end ▼
Use Zustand para salvar os itens e persistir via `localStorage`.
3
Dashboard de administração

0/4 concluídos
1

Rota protegida com layout próprio

2

Visão geral de vendas, pedidos e produtos

3
Gestão de produtos (CRUD)
Criar, editar, ativar e deletar produtos

Gestão de pedidos
