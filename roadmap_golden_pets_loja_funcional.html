Roadmap completo para colocar a Golden Pets no ar

1
Banco de dados e autenticação
Supabase — gratuito até 500MB · ~1 semana de trabalho
1
Criar conta e projeto no Supabase
Seu banco de dados PostgreSQL gratuito na nuvem
Grátis ▼
Acesse supabase.com e crie conta com GitHub
Clique em New Project → dê o nome "golden-pets" → escolha região South America (São Paulo)
Anote as credenciais: Project URL e anon public key — você vai precisar no .env
# instalar no seu projeto Next.js
npm install @supabase/supabase-js @supabase/ssr
Crie o arquivo .env.local na raiz do projeto com as variáveis do Supabase
O plano gratuito inclui 500MB de banco, 5GB de storage e 50.000 usuários autenticados — mais que suficiente para validar a loja.
2
Criar tabelas do banco de dados
Produtos, pedidos, clientes, categorias
Crítico ▼
No painel do Supabase vá em SQL Editor e execute as queries abaixo para criar as tabelas:
-- Tabela de produtos
CREATE TABLE products (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  name text NOT NULL,
  slug text UNIQUE NOT NULL,
  description text,
  price numeric(10,2) NOT NULL,
  original_price numeric(10,2),
  stock integer DEFAULT 0,
  category text,
  images text[],
  active boolean DEFAULT true,
  created_at timestamptz DEFAULT now()
);

-- Tabela de pedidos
CREATE TABLE orders (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  customer_email text NOT NULL,
  customer_name text,
  items jsonb NOT NULL,
  total numeric(10,2) NOT NULL,
  status text DEFAULT 'pending',
  payment_method text,
  payment_id text,
  address jsonb,
  created_at timestamptz DEFAULT now()
);
Crie também tabelas: customers (dados do cliente), categories (categorias de produtos) e admin_users (apenas você terá acesso).
3
Configurar autenticação de admin
Só você acessa o dashboard — nenhum cliente entra
Rápido ▼
No Supabase vá em Authentication → Users → Add user e crie seu usuário admin com email e senha forte
Crie o arquivo /app/admin/login/page.tsx com formulário de login usando supabase.auth.signInWithPassword()
Crie um middleware.ts na raiz do projeto para proteger todas as rotas /admin/* — redireciona para login se não autenticado
// middleware.ts — protege rotas admin
import { createMiddlewareClient } from '@supabase/ssr'
export const config = {
  matcher: ['/admin/:path*']
}
Isso garante que nenhum cliente acidentalmente acesse seu painel de gestão.
4
Conectar front-end ao banco
Substituir /data/products.ts pelo banco real
Crítico ▼
Crie /lib/supabase.ts com o client configurado
Nas suas páginas de produto, substitua o import estático por uma query real:
// Em /app/produtos/page.tsx
const { data: products } = await supabase
  .from('products')
  .select('*')
  .eq('active', true)
  .order('created_at', { ascending: false })
Como é Next.js com Server Components, essa query roda no servidor — o banco nunca fica exposto ao cliente.
2
Gateway de pagamento
MercadoPago — melhor opção para Brasil iniciante · ~1 semana
1
Por que MercadoPago para começar
Comparativo dos gateways para o mercado brasileiro
Info ▼
MercadoPago — Taxa ~4,49% (cartão 14 dias). Pix + boleto + cartão. API fácil, amplamente conhecido pelo consumidor BR. Ideal para começar.
Pagar.me — Taxa 3,99% + R$0,99. Checkout mais bonito, API mais técnica. Melhor para volume médio (acima de 50 pedidos/mês).
Stripe — Taxa 3,99% + R$0,39. Excelente documentação, suporta Pix. Indicado se você quiser vender para fora do Brasil também.
PagBank — Similar ao MercadoPago, popular no Brasil, checkout reconhecido.
Recomendação: comece com MercadoPago por ser o mais fácil de integrar, mais reconhecido pelo consumidor brasileiro e sem mensalidade. Quando tiver volume, migre para Pagar.me para pagar menos taxa.
2
Criar conta e obter credenciais
MercadoPago Developer — acesso à API
Grátis ▼
Acesse mercadopago.com.br e crie conta como pessoa física ou jurídica (PJ é obrigatório para sacar acima de R$20k/mês)
Vá em Seu negócio → Configurações → Credenciais para pegar o Access Token e Public Key
Use as credenciais de teste primeiro — o MercadoPago tem ambiente sandbox completo
npm install mercadopago
Adicione no .env.local: MP_ACCESS_TOKEN e NEXT_PUBLIC_MP_PUBLIC_KEY
3
Criar a rota de API de checkout
/api/checkout — recebe pedido e cria preferência de pagamento
Crítico ▼
Crie o arquivo /app/api/checkout/route.ts
// Recebe os itens do carrinho e cria preferência no MP
import MercadoPagoConfig, { Preference } from 'mercadopago'

export async function POST(req) {
  const { items, customer } = await req.json()
  const mp = new MercadoPagoConfig({
    accessToken: process.env.MP_ACCESS_TOKEN
  })
  const pref = await new Preference(mp).create({
    body: { items, payer: customer,
    back_urls: { success: '/pedido/sucesso' } }
  })
  return Response.json({ id: pref.id })
}
O MercadoPago redireciona o cliente para a página deles para pagar e depois volta para sua URL de sucesso. Simples e seguro.
4
Webhook — confirmar pagamento e salvar pedido
MercadoPago avisa seu site quando o pagamento é aprovado
Crítico ▼
Crie /app/api/webhook/route.ts — o MercadoPago vai chamar essa URL quando o pagamento for confirmado
Nessa rota: valide a assinatura do MP, busque os detalhes do pagamento pela API do MP, salve o pedido no Supabase com status "pago" e atualize o estoque
Configure a URL do webhook no painel do MercadoPago em Configurações → Notificações → Webhooks
Atenção: sem o webhook, você nunca saberá se o pagamento foi aprovado. É o passo mais crítico de toda a integração de pagamentos.
5
Carrinho de compras no front-end
Estado global com Zustand — persiste entre páginas
Front-end ▼
npm install zustand
Crie /store/cartStore.ts com Zustand para gerenciar os itens do carrinho
Estado do carrinho: lista de itens, total, adicionar, remover, limpar
Use localStorage persist do Zustand para o carrinho não sumir ao recarregar a página
Crie o componente CartDrawer.tsx — painel lateral que desliza com os itens e botão de finalizar compra
Zustand é simples de usar e se integra perfeitamente com Next.js. Muito mais fácil que Redux para este caso.
3
Dashboard de administração
Sua tela de gestão da loja — só você acessa · ~1-2 semanas
1
Estrutura do painel /admin
Rota protegida com layout próprio
Estrutura ▼
Crie o grupo de rotas /app/admin/ com layout próprio (sidebar + header) separado do layout público da loja
Estrutura de páginas do admin:
/app/admin/
  layout.tsx     ← sidebar + proteção de rota
  page.tsx       ← dashboard com métricas
  produtos/
    page.tsx     ← lista de produtos
    novo/page.tsx ← formulário de novo produto
    [id]/page.tsx ← editar produto
  pedidos/
    page.tsx     ← todos os pedidos
    [id]/page.tsx ← detalhe do pedido
  clientes/page.tsx ← lista de clientes
2
Tela de dashboard — métricas em tempo real
Visão geral de vendas, pedidos e produtos
Visual ▼
Cards de métricas no topo: Receita do mês, Pedidos hoje, Ticket médio, Produtos ativos
Gráfico de vendas dos últimos 30 dias (usar Recharts — já vem com o shadcn/ui)
Tabela dos 5 últimos pedidos com status colorido (pendente/pago/enviado/entregue)
Alertas: produtos com estoque baixo (abaixo de 5 unidades)
npm install recharts
3
Gestão de produtos — CRUD completo
Criar, editar, ativar e deletar produtos
Essencial ▼
Tabela paginada com todos os produtos: imagem, nome, preço, estoque, status ativo/inativo
Formulário de novo produto com os campos: nome, slug (auto-gerado), descrição, preço, preço original, estoque, categoria, imagens
Upload de imagens via Supabase Storage — drag-and-drop direto no formulário
Botão de ativar/desativar produto sem deletar (soft delete)
Use o componente shadcn/ui Form com react-hook-form e zod para validação — economiza muito tempo.
npm install react-hook-form zod @hookform/resolvers
4
Gestão de pedidos
Ver, filtrar e atualizar status de cada pedido
Essencial ▼
Tabela de pedidos com filtro por status: Pendente / Pago / Enviado / Entregue / Cancelado
Página individual do pedido: itens comprados, dados do cliente, endereço de entrega, método de pagamento, histórico de status
Botão para atualizar status manualmente (ex: marcar como enviado e adicionar código de rastreio)
Campo de código de rastreio — quando preenchido, o cliente recebe e-mail automático com o código
Para envio de e-mail, use Resend (gratuito até 3.000 e-mails/mês) + React Email para montar templates bonitos em JSX.
npm install resend react-email
4
Deploy e domínio
Colocar no ar de verdade · ~2 dias
1
Deploy na Vercel
Gratuito, deploy automático a cada push
Grátis ▼
Suba o código para o GitHub em um repositório privado
Acesse vercel.com, conecte com o GitHub e importe o repositório
Adicione todas as variáveis do .env.local no painel de Environment Variables da Vercel
A Vercel detecta Next.js automaticamente e configura o build sem precisar mexer em nada
Após o deploy, você terá uma URL como golden-pets.vercel.app para testar
A cada vez que você fizer push para a branch main, a Vercel faz deploy automático. Branches diferentes viram previews com URL própria.
2
Comprar domínio .com.br
goldenpets.com.br passa credibilidade
~R$ 40/ano ▼
Registre em registro.br (oficial) ou Hostinger (mais barato com hospedagem)
Após comprar, no DNS do domínio adicione um registro CNAME apontando para cname.vercel-dns.com
Na Vercel em Settings → Domains adicione o domínio personalizado
A Vercel gera o certificado SSL (HTTPS) automaticamente e de graça
Domínio .com.br transmite muito mais confiança para o consumidor brasileiro do que .vercel.app ou .com.
3
Configurar variáveis de ambiente de produção
Separar ambiente de teste do ambiente real
Crítico ▼
No MercadoPago: troque as credenciais de teste pelas de produção nas variáveis da Vercel
No Supabase: confirme que o projeto está com Row Level Security (RLS) ativado nas tabelas públicas
Configure as URLs de retorno do MercadoPago para o domínio real (não mais localhost)
Nunca coloque credenciais de produção no .env.local ou no GitHub. Use apenas nas variáveis de ambiente da Vercel.
5
Pré-lançamento e primeiras vendas
Checklist final antes de divulgar · ~3 dias
1
Páginas legais obrigatórias
Sem isso você não pode vender legalmente
Obrigatório ▼
Política de Privacidade — obrigatória pela LGPD. Descreve quais dados você coleta e como usa
Termos de Uso — regras de uso da loja, limitação de responsabilidade
Política de Trocas e Devoluções — o Código de Defesa do Consumidor garante 7 dias de arrependimento
Informações de entrega — prazo estimado, transportadoras utilizadas
Use o gerador gratuito de termos do Jusbrasil ou LegalFind como base — adapte com os dados da sua loja.
2
Testar o fluxo de compra completo
Do produto até o pedido salvo no banco
Não pule ▼
Use os cartões de teste do MercadoPago para simular uma compra aprovada, recusada e pendente
Confirme que o pedido aparece no banco de dados após o pagamento aprovado
Confirme que o e-mail de confirmação chega na caixa do cliente
Confirme que o dashboard admin mostra o pedido e que você consegue atualizar o status
Teste no celular — a maioria das compras no Brasil é feita pelo mobile
Se o webhook não funcionar em produção, nenhum pedido será registrado. Teste isso antes de divulgar.
